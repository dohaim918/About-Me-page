## 📋 Project Overview


이번 프로젝트는 자기소개 페이지를 넘어 **"디자인한 화면을 실제 코드로 어떻게 효율적으로 옮길 것인가?"** 라는 질문에서 시작되었습니다.
<br/>화면이 비어 보이지 않는 **균형감(Balance)**과 정보가 자연스럽게 읽히는 **흐름(Flow)**을 최우선으로 고려했습니다.

<br/>

### ​💚 Project Story​ 💚
> **"단순히 예쁜 화면이 아닌, 개발 가능한 구조를 설계하다."**

<p><b>Period</b> : `2025.12.12 ~ 12.18` (1 Week) </p>

<br/>

<p><b>Type</b> : PC First & Responsive Web </p>

<br/>

<p><b>Concept</b> : Soft Gradient & Clean Layout </p>

<br/>

<br/>

**​✳️ Goals ​✳️**  

<p>피그마(Figma) 디자인의 100% 구현</p>

## ✨ Key Features

| **Part** | **Description** |
|:---:|:---|
| **Layout** | **PC First 기준 설계.** 상단 네비게이션을 고정(Fixed)하여 접근성을 높이고, 스크롤 흐름에 따라 정보가 이어지도록 구성 |

## 🔥 Troubleshooting Journey
> **"반응형 레이아웃, 감이 아닌 논리로 해결하다."**

<details>
<summary><b>🚨 Issue: 유동적인 레이아웃 깨짐 현상 (Click)</b></summary>
<br/>

**[문제 상황]**
초기에는 유연한 화면을 위해 `vw`, `%` 단위를 주로 사용했으나, 화면 크기를 줄일 때 정렬이 어긋나거나 의도치 않은 위치로 밀려나는 현상이 발생했습니다.

**[해결 과정]**
1. **Flex 남용 자제**: 무조건적인 Flex 사용을 줄이고, `max-width`와 `width: 100%`를 기준으로 잡아 전체 컨테이너의 안정성을 먼저 확보했습니다.
2. **Auto-fit 도입**: 상단 이미지 영역 등은 `auto-fit`을 사용하여 공간 부족 시 자연스럽게 줄바꿈되도록 수정했습니다.
3. **Transition 적용**: 레이아웃이 변하는 분기점마다 `transition`을 주어 사용자가 느끼는 시각적 끊김을 최소화했습니다.

</details>
